========================================================
USER STORIES — SYSTEM ADMINISTRATOR
Project: American Computers Albania — Supply Chain AI
========================================================

ROLE DESCRIPTION:
The System Administrator manages user accounts, controls
access to the system, monitors system health, and maintains
the data pipeline and ML models.

--------------------------------------------------------
STORY 1 — User Management
--------------------------------------------------------
As a System Administrator,
I want to view and manage all registered users in the system
So that I can control who has access and what permissions they have.

Acceptance Criteria:
- User management page is only visible to admin role
- Table shows all users with ID, username, full name, email, role and status
- I can deactivate a user account with a single click
- Active and inactive users are shown with different status badges

--------------------------------------------------------
STORY 2 — User Registration Control
--------------------------------------------------------
As a System Administrator,
I want the first registered user to automatically become admin
So that there is always at least one admin to manage the system.

Acceptance Criteria:
- First registered user gets role = admin automatically
- All subsequent users get role = viewer by default
- Admin role cannot be removed from the only admin in the system

--------------------------------------------------------
STORY 3 — ETL Pipeline Monitoring
--------------------------------------------------------
As a System Administrator,
I want to trigger and monitor the ETL data pipeline
So that I can ensure the ML models are trained on fresh data.

Acceptance Criteria:
- ETL pipeline has 5 visible stages: ingestion, cleaning, normalization, feature engineering, export
- Each stage shows a status badge (Ready / Running / Done)
- Console output shows timestamped log messages
- Records processed, null rate and duration are shown after completion

--------------------------------------------------------
STORY 4 — API Health Monitoring
--------------------------------------------------------
As a System Administrator,
I want to see the API connection status in real time
So that I know immediately if the backend is down.

Acceptance Criteria:
- API status indicator is always visible in the sidebar
- Status checks automatically every 30 seconds
- Green dot = online, red dot = offline, amber dot = checking
- Status shows the API URL (localhost:8000)

--------------------------------------------------------
STORY 5 — Password Reset Management
--------------------------------------------------------
As a System Administrator,
I want users to be able to reset their passwords via email
So that they are not locked out of the system.

Acceptance Criteria:
- Password reset link is sent to the user's registered email via Resend API
- Reset tokens expire after a set time period
- Used tokens cannot be reused
- Reset password page validates new password before saving

--------------------------------------------------------
STORY 6 — Anomaly Detection Review
--------------------------------------------------------
As a System Administrator,
I want to review flagged anomalous orders in the system
So that I can investigate suspicious shipments and take action.


Acceptance Criteria:
- Anomaly detection scans recent orders from the database
- Each anomaly shows order reference, anomaly type and description
- Severity is shown as critical (red) or warning (amber)
- I can adjust the number of orders to scan using a slider
- Score distribution chart shows normal vs anomalous orders
